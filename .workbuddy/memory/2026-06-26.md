## 当量识别技术可行性评估（DDR-008）

### 用户质疑
用户提出关键问题："系统有办法识别未爆弹当量并且纳入系统考虑吗"——如果当量无法获取，殉爆安全链就是空中楼阁

### 架构师评估结论
1. **没有任何传感器能直接测量装药量**——所有路径都是间接的
2. **但数学站在我们这边**——殉爆公式 D=K×W^(1/3) 的立方根大幅压缩当量误差：
   - ±50% 当量误差 → 仅 ±14.5% 安全距离误差
   - 安全系数 K 本身含 1.5-3 倍余量，完全覆盖
3. **推荐分层降级策略**：
   - 第一优先：型号识别法（±5-15%）→ 查弹药数据库
   - 第二优先：多源融合法（±30-50%）→ 尺寸反推+磁力+类别先验
   - 第三兜底：最保守假设法 → 类别最大值×1.5
   - 并行通道：人工标注 → 操作员随时介入
4. **GPR 内部结构法直接放弃**——金属壳体对电磁波有屏蔽效应，物理定律决定
5. **F-030 数据库存在缺口**——存储的是"识别特征"不是"毁伤参数"，需扩展加入装药量字段
6. **DDR-007 原则不需要修正**，但需补充：安全链接收 YieldEstimate（当量+不确定度），数据缺失时 fail-safe 用保守默认值

### 新增数据结构
- YieldEstimate { nominalYield, uncertainty, confidence, estimationMethod }
- SafeDistanceResult { distance, marginApplied, confidence, effectiveYield }
- TargetInfo 需增加 yield 字段

### 涉密问题
- 装药量是军事情报，需分级存储、加密、审计追踪
- 安全链进程只接收脱敏数值（当量+不确定度），不接触型号等涉密信息
