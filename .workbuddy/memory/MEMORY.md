# 排弹抢修系统 项目约束条件（长期记忆）

## 硬件设计约束
- **全国产化**：所有设备须为中国自主研发生产，不依赖进口
- **不定品牌型号**：文档只定义功能需求，供应商选型是下一步
- **货架产品优先**：系统集成商不开发硬件，所有设备须为可采购的成熟产品
- **必须验证后才推荐**：任何方案须经市场调研确认产品存在，不能凭空设计

## 核心设计理念
- **无人化智能化**：全系统最小化人员在危险区域的暴露。空中探测无人化，地面处置遥控/自主化，目标从发现到排除全流程无需人员进入跑道危险区

## 部署约束
- **战时系统**：所有设备平时存仓库，战时拉出部署
- **跑道零固定设施**：无机巢、无永久建筑、无固定设备
- **部署时间 ≤20分钟**（仓库→跑道就绪）

## 通信约束
- **全封闭网络**：不依赖公网（4G/5G/互联网）
- **私有频段**：图传≥5GHz，Mesh≤1GHz，有线光纤精操作

## 无人机约束
- **单载荷多次起降**：一次只挂一种传感器
- **监控无人机全程在空**：从T+18起至任务结束
- **飞行顺序**：EO/IR → LiDAR → 磁力仪
- **LiDAR优先级低于EO/IR**：唯一不可替代价值是穿烟/穿尘
- **磁力仪机载**：效率远超地面，国产航磁成熟
- **GPR地面**：机载GPR不可行
- **激光测距仪不需要**

## 通信架构
- **MQTT统一消息总线**
- **空中中继替代地面中继主力**
- **机器人三层通信**：Mesh无线 + 有线光纤精操作 + 离线自主

## 机器人约束
- **GPR集成于机器人**：取消独立探测载车，探地雷达直接安装在排爆机器人上
- **单一机器人平台**：一台机器人同时承担GPR复察和排爆处置
- **GPR可收放**：机器人在移动时GPR天线收起不影响通过性，到达目标点后放下贴近地面扫描
