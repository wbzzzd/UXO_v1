## 核心问题

DDR-007 原则 1 要求殉爆安全链物理隔离。但殉爆安全距离公式是 D = K × W^(1/3)，其中 W 是装药当量。**如果系统无法获取 W，安全链就是空中楼阁。**

本文档回答：W 从哪里来？能不能来？来不了怎么办？

---

## 一、传感器能力 vs 当量识别需求

| 传感器 | 直接推算当量 | 型号识别辅助 | 尺寸测量 | 内部结构探测 | 综合价值 |
|--------|:---:|:---:|:---:|:---:|:---:|
| 红外热成像 | ❌ | ⚠️ 弱 | ❌ | ❌ | 低 |
| 3D激光扫描 | ⚠️ 间接(粗) | ✅ 强 | ✅ 高精度 | ❌ | 中高 |
| 广角相机 | ❌(需DB) | ✅ 最强 | ⚠️ 中等 | ❌ | 高 |
| 探地雷达GPR | ❌ | ❌ | ⚠️ 粗糙(埋地) | ❌ 金属屏蔽 | 低 |
| 磁力探测 | ⚠️ 很粗 | ❌ | ❌ | ❌ | 低 |

**关键发现：没有任何传感器能直接测量装药量。** 所有路径都是间接的。

## 二、8 条技术路径评估

| 路径 | 可行性 | 精度 | 推荐角色 |
|------|:---:|:---:|:---:|
| 1. 型号识别法（识别型号→查数据库） | 高 | ±5-15% | **主路径** |
| 2. 尺寸反推法（几何尺寸→估算装药体积） | 中 | ±40-60% | 辅助验证 |
| 3. GPR内部结构法 | **低** | 不可用 | **放弃**（金属屏蔽） |
| 4. 磁力特征法 | 低 | ±50-100% | 弱辅助 |
| 5. 多源融合法 | 中 | ±20-50% | **融合框架** |
| 6. 人工标注法 | 高 | 依赖人 | **兜底+修正** |
| 7. 最保守假设法（类别最大值×1.5） | 高 | 过保守 | **安全兜底** |
| 8. 情报关联法 | 中 | 依赖情报 | 先验输入 |

## 三、关键数学洞察：立方根的宽容性

殉爆公式 D = K × W^(1/3) 的立方根大幅压缩当量误差：

| 当量误差 | 安全距离误差 | 评价 |
|---------|:---:|------|
| ±5%（型号识别成功） | ±1.6% | 可忽略 |
| ±15% | ±4.8% | 可接受 |
| ±30% | ±9.1% | 可接受 |
| ±50%（尺寸反推） | ±14.5% | 需余量补偿 |
| ±100% | ±26.0% | 需较大余量 |

**即使当量误差高达 ±50%，安全距离误差只有 ±14.5%。** 安全系数 K 本身包含 1.5-3 倍余量，完全覆盖。这是当量识别"可行"的数学根基。

---

## 四、推荐策略：分层降级链

```
第一优先级：型号识别法
    ↓ 识别成功？
    ├─ 是 → W ± 5-15% → 安全距离 ± 1.6-4.8% → 标准余量 → 完成
    └─ 否 ↓

第二优先级：多源融合法（尺寸反推 + 磁力 + 类别先验）
    ↓ 融合估计？
    ├─ σ_W < 50% → W ± 30-50% → 加 2σ 余量 → 完成
    └─ 无数据 ↓

第三优先级：最保守假设法
    按类别最大装药量 × 1.5
    → 安全距离偏大 2-3 倍 → 大余量 → 完成（安全但效率低）

并行通道：人工标注
    操作员可在任何阶段介入，人工输入优先级 > 自动估计
```

**降级链的关键特性**：
1. 永远能给出答案——不会因识别失败而安全链瘫痪
2. 精度越高，处置区域越小——激励提升识别能力
3. 不确定度始终透明——操作员知道当前安全距离的置信度
4. 人始终在回路中——可基于情报做最终判断

## 五、弹药数据库需求

F-030（目标特征数据库）**不覆盖**当量需求。它存储的是"识别特征"（这是什么弹），不是"毁伤参数"（这弹多大威力）。

需要扩展 F-030 或新增弹药毁伤参数数据库，关键字段：
- charge_weight_nominal（标称装药量）
- charge_weight_min/max（批次差异范围）
- explosive_type/density（装药类型和密度）
- tnt_equivalent_ratio（TNT当量换算系数）
- wall_thickness_typical/range（壁厚参数）
- 各类安全系数 K

涉密问题：装药量是军事情报，需分级存储、加密、审计追踪。安全链进程只接收脱敏后的数值（当量+不确定度），不接触型号等涉密信息。

## 六、对 DDR-007 的影响

**DDR-007 的 5 条原则不需要修正，但需要补充落地实现：**

| 原则 | 影响 | 补充 |
|------|------|------|
| 1. 安全链物理隔离 | 不变 | 安全链接收 YieldEstimate 纯数据输入 |
| 2. 故障即安全 | 补充 | 当量数据缺失=故障，默认使用保守当量 |
| 3. 数据通道分级 | 不变 | 当量估计走 MQTT 控制通道 |
| 4. 外部依赖零信任 | 补充 | 弹药数据库失效时，安全链用内置保守默认值表 |
| 5. 可独立验证 | 不变 | 安全链测试用固定 YieldEstimate 验证 |

### 新增数据结构

```cpp
struct YieldEstimate {
    double nominalYield;         // 标称当量 (kg TNT)
    double uncertainty;          // 不确定度 (kg, 1σ)
    YieldConfidence confidence;  // HIGH/MEDIUM/LOW/CONSERVATIVE
    QString estimationMethod;    // MODEL_ID/DIMENSION/FUSION/MANUAL/CONSERVATIVE
};

struct SafeDistanceResult {
    double distance;             // 安全距离 (m)
    double marginApplied;        // 应用的余量系数
    YieldConfidence confidence;  // 置信度
    double effectiveYield;       // 实际用于计算的当量（含余量）
};
```

### 对现有代码的影响

| 影响项 | 需要的变更 |
|--------|----------|
| TargetInfo | 增加 YieldEstimate yield 字段 |
| PathPlanner::calculateSafeDistance() | 改为接收 YieldEstimate，返回 SafeDistanceResult |
| F-030 数据库 | 扩展加入装药量等毁伤参数 |
| SafetyChainModule | 增加当量输入接口和保守降级逻辑 |
| AI 识别输出 | 增加当量估计+不确定度输出 |
| UI | 增加当量信息展示和手动修正界面 |

## 七、最终结论

**当量识别在当前技术条件下可行，但不是"测量"，是"估计+降级+余量"。**

1. 没有传感器能直接测量装药量，但型号识别法可达 ±5-15% 精度
2. 立方根公式天然宽容当量误差——±50% 当量误差只导致 ±14.5% 安全距离误差
3. 识别全失败时按类别最大装药量保守假设，安全距离偏大但安全
4. 操作员始终可手动介入
5. 安全链接收"当量+不确定度"，对不确定度做余量补偿
6. 弹药数据库需扩展，涉密数据需分级处理

**殉爆安全链在当量识别方面可行，DDR-007 的原则不需要推翻，但需要在数据结构、数据库、降级策略三个层面做具体补充。**
