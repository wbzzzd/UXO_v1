# 架构边界

最后更新：2026-07-07

## 当前边界

- `src/App/` 和 `include/App/`：应用生命周期、配置路径和初始化流程。
- `src/MainWindow/` 和 `include/MainWindow/`：当前可见界面骨架和主窗口区域。
- `include/Core/Data/Types.h`：当前共享数据类型入口。
- `include/Core/Simulation/` 和 `src/Core/Simulation/`：当前本地模拟演示数据入口。
- `config/`：本地配置和后续模拟数据输入位置。

## 未来边界建议

- `Core`：放稳定的数据模型、应用服务接口和纯逻辑，不直接依赖具体 UI 控件。
- `MainWindow`：只负责界面组合、用户交互和展示状态，不直接控制真实设备。
- `Modules`：后续按态势、设备、决策等业务能力拆分，先服务模拟流程。
- `Drivers`：真实设备驱动或协议适配只能在用户明确授权后引入，并必须和模拟实现隔离。
- `Storage`：数据库写入不是 MVP 前置条件，先以内存状态或本地只读配置支撑演示。

## 设计纪律

- 不为尚未实现的模块创建大而空的目录树。
- 不把文档中的目标模块写成当前事实。
- 新增边界前必须有调用方或演示流程需要。
- 接口占位必须明确标注模拟、占位或未来真实接入。
