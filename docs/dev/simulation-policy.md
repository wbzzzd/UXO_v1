# 模拟策略

最后更新：2026-07-07

## 基本原则

排弹抢修系统涉及真实设备和安全关键动作。当前阶段默认只允许本地模拟、只读分析和接口占位。

## 允许

- 使用本地 JSON 或内存对象模拟任务、目标、设备状态和处置方案。
- 在 UI 中展示模拟设备在线、电量、位置、工具状态和任务进度。
- 为未来真实接口定义边界清晰的抽象，但不连接真实设备。
- 在日志、标题或字段中明确标注模拟来源。

## 禁止

- 发送真实设备控制命令。
- 执行真实排弹或抢修动作。
- 将模拟数据写入真实数据库或外部系统。
- 在 UI 或文档中把模拟状态描述成真实接入状态。
- 未经用户明确授权接入外部通信链路。

## 命名建议

- 模拟服务使用 `Mock`、`Simulation` 或 `Demo` 前缀/后缀。
- 真实接入接口使用 `Interface` 或 `Adapter`，并在文档中说明尚未接入。
- UI 文案应使用“模拟任务”“模拟设备”“演示状态”等明确措辞。
