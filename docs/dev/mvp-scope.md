# MVP 范围

最后更新：2026-07-07

## MVP 目标

MVP 应先做一个可演示、可理解、风险可控的指挥席闭环：

- 程序启动后显示主窗口。
- 使用本地模拟数据展示任务、目标、设备状态和处置方案。
- 支持最小流程：查看态势、查看设备、查看或选择处置方案、记录模拟状态变化。
- 所有外部设备、通信、数据库和算法能力先用接口占位或模拟数据表达。

## MVP 非目标

- 不接真实设备。
- 不发送真实排弹、机械臂、无人机或传感器控制命令。
- 不实现安全关键自动决策。
- 不把模拟数据写成真实状态。
- 不一次性补齐 SRS/SDD 中的全部模块。

## 优先顺序

1. 启动可见主窗口。
2. 统一最小数据模型和模拟数据来源。`Core::Simulation::DemoScenarioProvider` 已完成最小本地模拟场景。
3. 打通态势、设备、决策三块 UI 的本地数据流。
4. 建立演示模式和操作日志。
5. 在文档和接口中保留真实接入边界。

## 验收原则

MVP 的验收重点是流程闭环和边界清晰，而不是功能数量。任何看似真实设备控制的行为都必须标为模拟或占位。
